# Database Schema

PostgreSQL 15. All migrations managed by Flyway under `backend/src/main/resources/db/migration/`.

---

## Tables

### devices
Registered edge hosts. Written on first contact; updated on each telemetry batch.

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | PK, generated by backend |
| `hostname` | TEXT | Unique, not null |
| `os_info` | TEXT | e.g. "Linux 6.1.0 x86_64" |
| `agent_version` | TEXT | Edge client version string |
| `registered_at` | TIMESTAMPTZ | Default now() |
| `last_seen_at` | TIMESTAMPTZ | Updated on every ingest |

---

### telemetry_snapshots
One row per collection cycle per device. High write volume; queries always scoped to a device + time window.

| Column | Type | Notes |
|--------|------|-------|
| `id` | BIGSERIAL | PK |
| `device_id` | UUID | FK → devices.id |
| `captured_at` | TIMESTAMPTZ | Timestamp from edge client |
| `cpu_pct` | NUMERIC(5,2) | 0.00 – 100.00 |
| `mem_used_mb` | INTEGER | |
| `mem_total_mb` | INTEGER | |
| `disk_used_gb` | NUMERIC(8,2) | |
| `disk_total_gb` | NUMERIC(8,2) | |
| `load_avg_1m` | NUMERIC(6,3) | Unix load average |

**Indexes**: `(device_id, captured_at DESC)` for time-range queries.

---

### fault_events
Crash and fault events detected by the edge client.

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | PK |
| `device_id` | UUID | FK → devices.id |
| `occurred_at` | TIMESTAMPTZ | When the fault happened on the host |
| `fault_type` | TEXT | SIGSEGV \| SIGABRT \| SIGBUS \| OOM \| PROCESS_EXIT \| TIMEOUT \| UNKNOWN |
| `process_name` | TEXT | Name of the affected process |
| `exit_code` | INTEGER | Process exit code if available |
| `raw_log` | TEXT | Full crash log or stack trace |
| `status` | TEXT | pending \| processing \| done \| failed |
| `processing_started_at` | TIMESTAMPTZ | Set when triage agent claims the event; used by watchdog to detect stale jobs |
| `created_at` | TIMESTAMPTZ | Default now() |

**Indexes**: `(status)` for triage agent polling, `(device_id)` for UI queries.

**Status lifecycle**:
```
           ┌─────────────┐
           │   pending   │◄──── watchdog resets stale events here
           └──────┬──────┘
                  │ triage agent claims event
           ┌──────▼──────┐
           │ processing  │
           └──────┬──────┘
          ┌───────┴───────┐
          ▼               ▼
       ┌──────┐       ┌────────┐
       │ done │       │ failed │
       └──────┘       └────────┘
```

---

### triage_reports
One report per fault event. Written by the Python triage agent.

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | PK |
| `fault_event_id` | UUID | FK → fault_events.id, UNIQUE |
| `analyzed_at` | TIMESTAMPTZ | Default now() |
| `root_cause` | TEXT | Human-readable root cause summary |
| `confidence` | TEXT | high \| medium \| low |
| `affected_module` | TEXT | File, module, or component name |
| `recommendation` | TEXT | Suggested fix or investigation step |
| `raw_analysis` | JSONB | Full structured output from the triage agent |

**`raw_analysis` example**:
```json
{
  "signalName": "SIGSEGV",
  "faultAddress": "0x0000000000000000",
  "topFrame": "parse_frame",
  "matchedPattern": "null_deref",
  "stackDepth": 8,
  "patternVersion": "1.0"
}
```
